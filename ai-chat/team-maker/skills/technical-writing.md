# Technical Writing

## Principles
- Lead with the answer, not the reasoning
- One idea per paragraph
- Use concrete examples over abstract descriptions
- Prefer active voice: "the server validates" not "validation is performed by"

## Documentation Types

### README
- What it does (1 sentence)
- How to run it
- Configuration reference
- That's it

### API Docs
- Endpoint, method, auth requirements
- Request body with example
- Response body with example
- Error codes

### Architecture Docs
- System diagram (what connects to what)
- Data flow (how a request moves through the system)
- Key decisions and why they were made

## Formatting
- Use headers to enable scanning
- Code blocks with language tags
- Tables for reference data
- Bullet lists for related items, numbered lists for sequences
